Part 2 – 60 % - IT Support CRUD system and video
In the labs we build a simple key/value lookup CRUD system. For this coursework you must adapt
the same system to fit a small business scenario: a Helpdesk Response Library for a company. Each
MongoDB entry should store a Key (e.g. issue code) and a Value (the standard response text).
Example:
• 	Key: PWD_RESET has the value: “Ask the user to use the reset link and confirm their email
address.”
• 	Key: BILLING_LATE has the value: “Advise of the grace period and provide the payment
link.”
We will develop a basic system in the labs, and you will be shown how to evidence this for your
video. Higher marks will be awarded for:
Implementing the test page (a short staff training quiz based on your helpdesk entries, e.g. match
an issue code to the correct response)
Adding a new menu item and page.
Adding a new field to the CRUD system, the current system has 2, add a 3rd field (e.g.
“Category/Department”, “Priority”, or “Notes”).
Additional functionality of your own choice beyond the above three.
To be discussed more detail in the lectures.
